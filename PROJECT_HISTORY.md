# Todo App 작업 기록

이 문서는 `todo-app-composition` 프로젝트를 현재 대화 기준으로 정리한 작업 기록이다.
정확성 기준은 다음 두 가지다.

- 이 대화에서 실제로 주고받은 내용
- 현재 워크스페이스에서 확인한 파일 상태

대화 밖에서 별도로 진행된 작업은 추정하지 않았다.

## 프로젝트 개요

- 프로젝트명: `todo-app-composition`
- 목적: Vue 3 Composition API 기반 투두앱 구현
- 저장 방식: 브라우저 `localStorage`
- 주요 기능:
  - 할 일 추가
  - 할 일 완료/미완료 토글
  - 할 일 수정
  - 할 일 삭제
  - 전체 / 완료 탭 필터링
  - 완료 개수 / 남은 개수 표시
  - 완료 항목을 아래로 정렬

## 작업 흐름 요약

처음 확인한 상태에서는 컴포넌트 분리는 되어 있었지만, 반응형 상태 관리와 `localStorage` 연동 방식이 섞여 있어 수정 후 화면 갱신이 기대대로 동작하지 않는 부분이 있었다. 이후 상태 관리 흐름을 정리하고, 수정 UI를 안정화하고, 전체 디자인을 개선하는 방향으로 작업이 이어졌다.

## 사용자께서 진행한 작업

대화와 현재 파일 상태 기준으로 확인 가능한 내용만 정리했다.

1. `todo-app-composition` 프로젝트를 생성하고 기본 Vue/Vite 구조를 준비했다.
2. `App.vue`, `TodoList.vue`, `TodoInput.vue`, `TodoHeader.vue`, `useLocalStorage.js`를 사용해 투두앱 구조를 직접 구현했다.
3. `localStorage` 저장/복원 기능을 붙이려는 방향으로 코드를 확장했다.
4. `TodoList.vue`에서 수정 input 노출 토글과 재렌더링 문제를 직접 실험하며 코드를 변경했다.
5. UI가 좁게 보이는 문제를 해결하려고 `main.css`의 너비 관련 값을 직접 수정해 보았다.
6. 컴포넌트별로 Composition API 형태로 통일하는 방향을 원했고, 그 과정에서 `TodoHeader.vue` 변환을 요청했다.
7. 완료된 항목이 목록 아래로 정렬되도록 요구사항을 추가했다.

## 제가 진행한 작업

### 1. 반응형 상태 흐름 정리

초기 상태에서 부모 컴포넌트의 `todos` 계산 방식과 자식 컴포넌트의 props 처리 방식 때문에, 목록 수정 후 화면이 바로 갱신되지 않는 문제가 있었다. 이를 해결하기 위해 다음과 같이 정리했다.

- `todos`를 `localStorage`에서 초기 1회만 읽어오도록 수정
- `computedTodos`는 상태를 덮어쓰지 않고 필터링/정렬만 수행하도록 변경
- 추가/삭제/완료/수정 시에는 항상 `todos.value`를 직접 갱신하도록 정리
- `localStorage`는 저장소 역할만 수행하도록 분리

### 2. `useLocalStorage.js` 단순화

저장 유틸은 읽기/쓰기 역할만 하도록 줄였다.

- `getTodosFromLocalStorage()`: `localStorage`에서 읽고, 없으면 빈 배열 반환
- `setTodoLocalStorage(todos)`: 배열을 JSON 문자열로 저장

이렇게 분리해서 UI 정렬이나 필터링 로직은 저장 유틸에 넣지 않고, 화면 쪽에서 처리하게 유지했다.

### 3. 수정 UI 토글 구조 정리

`TodoList.vue`에서 수정 input을 토글하기 위해 각 항목별 상태 대신 `editingId`를 사용하는 구조를 제안하고 적용 방향을 설명했다.

- 현재 수정 중인 항목의 `id`를 `editingId`로 저장
- `editingId === item.id`일 때만 input 노출
- 저장 완료 후 `editingId`를 `null`로 되돌려 수정 모드 종료
- `newMsg`는 수정용 임시 입력 상태로 관리

이 구조로 "어떤 항목이 수정 중인지"를 명확하게 구분할 수 있게 됐다.

### 4. `TodoHeader.vue`를 Composition API로 변환

`TodoHeader.vue`는 Options API에서 `script setup` 기반 Composition API로 변환했다.

- `props`는 `defineProps`
- 이벤트는 `defineEmits`
- `updateTab()`은 `emit('update-tab', tab)` 형태로 단순화

이로써 현재 주요 컴포넌트들이 비슷한 스타일로 정리됐다.

### 5. 완료 항목 정렬 추가

목록 화면에서는 완료되지 않은 항목이 위에, 완료된 항목이 아래에 오도록 `computedTodos`에서 정렬을 추가했다.

- 원본 저장 구조를 바꾸지 않음
- 화면 렌더링 직전에만 정렬 적용
- `전체` 탭에서는 미완료 우선
- `완료` 탭에서는 완료 항목만 표시

### 6. UI 디자인 개선

전체 화면을 더 깔끔하고 세련된 카드형 레이아웃으로 재구성했다.

- 배경을 그라디언트 기반으로 변경
- 메인 패널을 유리감 있는 카드 레이아웃으로 변경
- 상단 헤더에 요약 카드 추가
- 탭 디자인 개선
- 리스트 아이템, 체크박스, 수정 input, 액션 버튼 스타일 정리
- 입력 영역과 버튼을 더 정돈된 형태로 개선
- 모바일 반응형 레이아웃 추가

### 7. 레이아웃 폭 문제 원인 파악 및 수정

`.todo` 자체의 `max-width`만 바꾸어도 폭이 넓어지지 않는 이유를 확인했고, 실제 원인이 상위 요소 `#app`의 너비 미설정이라는 점을 찾아 수정했다.

- `.todo`에 `max-width: 720px` 적용
- `#app { width: 100%; display: flex; justify-content: center; }` 추가

이후 카드가 의도한 최대 폭까지 정상적으로 넓어질 수 있게 됐다.

## 파일별 현재 역할

- `src/App.vue`
  - 전체 상태 관리
  - `localStorage` 초기 데이터 로드
  - 필터링, 정렬, 카운트 계산
  - 하위 컴포넌트 이벤트 처리
- `src/components/TodoHeader.vue`
  - 상단 제목, 요약 정보, 탭 전환 UI
- `src/components/TodoInput.vue`
  - 입력 상태 관리 및 할 일 추가 이벤트 발생
- `src/components/TodoList.vue`
  - 목록 출력
  - 완료 토글 / 수정 / 삭제 이벤트 발생
  - 수정 input 토글 상태 관리
- `src/hooks/useLocalStorage.js`
  - 투두 데이터 읽기 / 저장
- `src/assets/main.css`
  - 전반적인 앱 UI 스타일

## 현재 구조에서의 설계 기준

이 프로젝트는 다음 기준으로 역할을 나누는 것이 가장 자연스럽다.

- 저장소 역할: `useLocalStorage.js`
- 화면용 상태 가공: `App.vue`
- UI 상호작용: 각 컴포넌트

즉, 저장 유틸은 읽기/쓰기만 담당하고, 정렬/필터링/카운트 같은 UI 로직은 `App.vue`의 `computed`에서 처리하는 구조다.

## 현재 확인된 기능 목록

- `Enter` 또는 버튼 클릭으로 할 일 추가
- 체크박스로 완료 상태 변경
- 수정 아이콘 클릭 시 수정 모드 전환
- 수정 후 Enter 또는 체크 아이콘으로 저장
- 삭제 아이콘 클릭 시 항목 삭제
- `localStorage` 기반 새로고침 후 데이터 유지
- 상단 탭으로 전체 / 완료 목록 전환
- 상단 요약 카드에 전체 / 완료 / 진행중 개수 표시
- 완료 항목 자동 하단 정렬
